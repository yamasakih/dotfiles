# 目視確認チェックリスト

このコマンドは、コード変更後に目視で確認すべきポイントをユーザーに提案します。

## 使用方法

```
/visual-check
```

## このコマンドは以下の手順を実行します

1. 直前の会話で行った変更内容を分析します
2. 変更の種類に応じて、目視確認が必要な項目をリストアップします
3. 確認手順や確認コマンドも併せて提案します

## 確認すべき観点

### データベース関連の変更がある場合
- [ ] マイグレーションの実行が必要か（`npx prisma db push` または `npx prisma migrate dev`）
- [ ] 既存データへの影響はないか
- [ ] `npx prisma studio` でデータが正しく保存されているか

### API エンドポイントの変更がある場合
- [ ] リクエスト/レスポンスの形式が期待通りか
- [ ] エラーハンドリングが適切か
- [ ] 認証・認可が正しく機能しているか

### UI/フロントエンドの変更がある場合
- [ ] 画面表示が期待通りか
- [ ] レスポンシブ対応は問題ないか
- [ ] ユーザーインタラクション（クリック、入力など）が正しく動作するか
- [ ] エラー表示やローディング状態が適切か

### 通知・メール機能の変更がある場合
- [ ] 通知が正しい宛先に届くか
- [ ] 通知内容（タイトル、メッセージ、リンク）が正しいか

### 認証・セキュリティ関連の変更がある場合
- [ ] ログイン/ログアウトが正しく動作するか
- [ ] 権限のないユーザーがアクセスできないか
- [ ] セッション管理が適切か

## 出力形式

変更内容に応じて、以下の形式で提案してください：

```
## 目視確認チェックリスト

### 必須確認項目
1. [具体的な確認項目]
   - 確認手順: [手順]
   - 確認コマンド: `[コマンド]`（該当する場合）

### 推奨確認項目
1. [具体的な確認項目]

### 注意事項
- [特に注意すべき点があれば記載]
```

## 重要な注意事項

- 変更内容を正確に把握した上で、関連する確認項目のみを提案すること
- 確認手順は具体的かつ実行可能なものにすること
- 本番環境への影響がある場合は特に強調すること
- テストで網羅できない部分（UI表示、ユーザー体験など）を中心に提案すること
