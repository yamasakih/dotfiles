# Pull Request のレビューを行ってください。

Pull Request $ARGUMENTS のレビューを行い、結果をターミナルに出力しつつ、同じ内容を Pull Request にコメントとして投稿してください。

## 手順

1. **PR情報の取得**: `gh pr view` と `gh pr diff` でPRの詳細と差分を確認
2. **レビュー観点**:
   - PR説明と実際の変更内容が一致しているか
   - コードの品質（可読性、保守性、パフォーマンス）
   - バグや潜在的な問題がないか
   - テストが適切に書かれているか（該当する場合）
   - セキュリティ上の懸念がないか
3. **レビュー結果の作成**: 以下の形式でレビューコメントを作成
   - 問題点があれば指摘（重大度: 高/中/低）
   - 良い点があれば言及
   - 最終判断: Approve / Request Changes / Comment
4. **出力**: レビュー結果をターミナルに出力
5. **PRへの投稿**: `gh pr comment` で同じ内容をPRに投稿

## レビューコメントのフォーマット例

```markdown
## PR #XX レビュー

### [Approve / Request Changes / Comment]

---

#### 問題点（該当する場合）

**重大度**: 高/中/低
- 問題の説明
- 推奨される修正方法

---

#### 良い点

- ポジティブなフィードバック

---

[最終コメント]
```

## 注意事項

- 客観的かつ建設的なフィードバックを心がける
- 問題点だけでなく、良い点も言及する
- 具体的な改善案を提示する
- 日本語でレビューを行う
