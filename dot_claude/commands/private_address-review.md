# Pull Request のレビュー指摘に対応してください。

Pull Request $ARGUMENTS のレビュー指摘に reviewee として対応し、結果をターミナルに出力しつつ、同じ内容を Pull Request にコメントとして投稿してください。

## 手順

1. **レビューコメントの確認**: `gh pr view` でPRのレビューコメントを確認
2. **指摘内容の分類**:
   - **重大度 低**: `.claude/docs/issue.md` に「将来対応予定」として追記
   - **重大度 中以上**: 実際にコード修正や対応を行う
3. **対応の実施**:
   - 重大度 低: issue.md の適切なセクションに Improvement として追加
   - 重大度 中以上: 指摘に従ってファイルを修正し、コミット
4. **対応結果の作成**: 以下の形式で対応結果をまとめる
5. **出力**: 対応結果をターミナルに出力
6. **PRへの投稿**: `gh pr comment` で同じ内容をPRに投稿

## 対応結果のフォーマット

```markdown
## レビュー指摘への対応

### 1. [指摘内容の要約]

**重大度**: 高/中/低
**対応**: [対応済み / issue.mdに追加]

[対応内容の説明]

### 2. [次の指摘...]

...

---

ご確認をお願いします。
```

## 重大度の判断基準

| 重大度 | 基準 | 対応方法 |
|--------|------|---------|
| 高 | バグ、セキュリティ問題、機能が動作しない | 即座に修正 |
| 中 | コード品質、パフォーマンス、ベストプラクティス違反 | 即座に修正 |
| 低 | リファクタリング提案、将来の改善案、ドキュメント改善 | issue.md に追加 |

## issue.md への追加フォーマット

重大度「低」の指摘は以下の形式で `.claude/docs/issue.md` に追加：

```markdown
### Improvement #N: [タイトル] (PR #XX レビュー指摘)

**対象ファイル**:
- `path/to/file.ts`

**現状**:
[現在の問題点]

**改善案**:
[推奨される改善方法]

**優先度**: 低

---
```

## 注意事項

- 修正後は必ずリント・フォーマットチェックを実行
- コミットメッセージは変更内容を明確に記載
- 対応が難しい場合は理由を説明し、代替案を提示
- 日本語で対応結果を記載
